# DEVNOTES 2026-05-05T133000Z

This change addresses the startup lag reported for face detection, MediaPipe, and camera activation. The main cause was the display camera path waiting on script/model setup plus a 4-second minimum loading floor before the UI could settle. The fix preloads MediaPipe scripts and model constructors on mount, runs FaceMesh and Hands initialization in parallel, shortens the loading dwell to a brief 900ms, and keeps the water-mist button disabled until the camera stream is actually live.

It also adds a macOS `Start-InkLumina.command` launcher so release users can start the app by double-clicking, matching the existing Windows `.bat` flow.

Impact notes:

- Camera startup should feel much closer to the protected `static/display.html` flow because the expensive scripts begin loading before the user clicks the camera button.
- Water overlay interactions are now gated by live camera state instead of merely being visually present.
- The launch instructions in `README.md` now cover both Windows and macOS release users.
