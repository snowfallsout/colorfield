# DEVNOTES_2026-05-02T215000Z

## 背景

目前 server-side socket 邏輯分散在兩個檔案中：

- `src/lib/server/socket.server.ts`
- `src/lib/server/vite-socket-plugin.ts`

兩者都各自維護：

- `mbti:init` 初始 payload
- `state` 初始 payload
- `submit_mbti` 的 count / lucky / spawn 流程

這會造成 dev / prod 行為在後續修改時容易漂移。

## 本輪決策

### A. 抽出 shared server-side socket helper

新增：

- `src/lib/server/socket.shared.ts`

讓 dev plugin 與 production attach path 共用同一份 Socket.IO wiring。

### B. 保留兩個 entrypoint，但不再複製事件流程

- `socket.server.ts` 只保留 production attach 入口
- `vite-socket-plugin.ts` 只保留 Vite dev plugin 入口

兩者都改為呼叫 shared helper。

### C. 補清楚頂層 `services/mediapipe.ts` 的角色註解

它是通用 browser MediaPipe client service，供 `state/media.svelte.ts` 使用；不是 display route 專用的 camera owner。

## 驗證方式

本輪完成後需確認：

- `npm run check` 通過
- `socket.server.ts` 與 `vite-socket-plugin.ts` 不再各自維護重複 socket 事件流程
- `services/mediapipe.ts` 的角色說明更明確
